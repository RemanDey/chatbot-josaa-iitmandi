# Core-to-Software Transition Guide

Purpose: realistic guidance for students from core engineering branches who plan to move into software careers.

Summary
- Easiest transition: `CSE`, `DSE` (native). 
- Moderate transition: `ECE`, `EE`, `Instrumentation` (good hardware + systems overlap). 
- Harder transition: `ME`, `CE`, `Chemical` (requires intentional coursework, projects, and coding practice).

Key levers for transition
- DSA and systems practice: essential for product/software roles. Expect 6–12 months of focused practice to reach average placement readiness from non-CS backgrounds.
- Projects and internships: build portfolio with practical code, full-stack prototypes, or systems-level projects showing impact.
- Course selection: prioritize programming, OS, databases, networks, ML electives where available.

Branch-specific notes
- ECE/EE: many students become software engineers by focusing on embedded, signal-processing, or communications projects that include coding. Take software electives early.
- ME/CE: emphasize control, robotics, CAD+automation, and data-focused projects. Interview prep must cover DSA and systems design.
- Chemical/Civil: target domain-specific software roles (process automation, simulation, CAD/CAE tooling) or learn full-stack/ML skills for general software roles.

Recruiter eligibility patterns (realistic)
- Many tech recruiters accept strong portfolios from non-CS branches, but the signal must be explicit: internships, GitHub, and algorithmic readiness.
- Product-first companies often prefer CS graduates but hire exceptional non-CS candidates with demonstrable coding and product impact.

DSA grind expectations
- Non-CS students should plan deliberate DSA schedules: basics (2 months), intermediate (3 months), mock interviews & system design (3 months). Timelines vary by starting ability.
- Recommend problem categories: arrays/strings, trees/graphs, dynamic programming, hashing, greedy, and basic system design.

Practical recipe (timeline)
1. First 6 months: structured learning (programming + DSA fundamentals) + one team project.
2. Months 6–12: internships, open-source contributions, competitive mocks, and resume refinement.
3. Ongoing: targeted system-design study for senior roles; maintain portfolio.

Agent use notes
- If user is non-CS and placement-focused, recommend a combined plan: branch electives + explicit 9–12 month DSA track.

End.
# Core to Software Transition at IIT Mandi

**Overview:**
- Many students from core branches (EE, ME, CE, EP) at IIT Mandi aspire to move into software roles.
- The transition is possible, but requires significant self-study, coding practice, and resume building.

**Key Points:**
- Most software companies allow CSE, DSE, and EE students. Some allow ME/EP; few allow CE/BS Chem.
- The earlier you start coding (1st/2nd year), the better your chances.
- DSA (Data Structures & Algorithms) and coding contests (LeetCode, Codeforces) are essential.
- Build a strong GitHub/project portfolio and seek software internships.
- Network with seniors who have made the transition.

**Branch-wise Difficulty:**
- Easiest: CSE/DSE > EE > ME/EP > CE/BS Chem

**Summary:**
- Software transition is possible from any branch at IIT Mandi, but is easiest from circuital branches and hardest from non-circuital/core branches.
